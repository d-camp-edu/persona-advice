# 당뇨 처방 시뮬레이터 — 제약사 자문의 도구 기획안 (완전 스펙)

> 제약사 담당자/자문의가 의사에게 당뇨 약제 처방 시나리오를 1:1로 시연하는 웹 앱.
> 병원명 + 의사이름으로 입력 → 환자 1명 선택 → 처방 → 결과 리포트 확인.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 명칭 | Persona Rx (가칭) |
| 목적 | 의사에게 당뇨 약제 처방 시나리오를 실시간 시뮬레이션으로 시연 |
| 사용자 | 제약사 담당자(시연 진행) + 의사(처방 체험) |
| 플랫폼 | 모바일 우선 웹앱 (max-width: 640px) |
| 스택 | React + Vite + Tailwind CSS + Firebase Firestore |

---

## 2. 화면 흐름

```
[1. 로그인]
  병원명 + 선생님 이름 입력 → 시작
        ↓
[2. 환자 선택]
  공병증 필터 → 환자 카드 선택
        ↓
[3. 처방 진행]
  환자 차트 확인 → 5슬롯 약제 선택 → 상병코드 선택 → 처방 확정
        ↓
[4. 결과 리포트]
  지표 변화 / 공병증 반응 / 부작용 / 보험 삭감 / 환자 피드백
        ↓
  [다른 환자 시뮬레이션] ──→ [2. 환자 선택]
  [처음으로] ─────────────→ [1. 로그인]
```

## 3. 화면별 상세 스펙

---

### 3-1. 로그인 화면

**레이아웃**: 중앙 정렬 카드, 배경 그라데이션 (인디고→퍼플, Admin에서 변경 가능)

**입력 필드**:
- 병원명 (text input, placeholder: "예: 서울대학교병원 내분비내과")
- 선생님 이름 (text input, placeholder: "예: 홍길동")

**버튼**: "시뮬레이션 시작하기" → 두 필드 모두 입력 시에만 활성화

**세션 키**: `${병원명}_${선생님이름}` → Firebase 문서 ID로 사용 (특수문자 제거)

**관리자 접근**: 우하단 설정 아이콘 → 비밀번호(Admin에서 설정) 입력 → Admin 화면

---

### 3-2. 환자 선택 화면

**헤더**: "{선생님이름} 선생님, 어떤 환자를 시뮬레이션 해볼까요?"

**공병증 필터 탭** (수평 스크롤):
- 전체 / 심부전 / 만성신장질환(CKD) / 비만 / 심혈관 / MASH / Stroke / 위장장애 / 생식기감염 / 저혈당

**환자 카드** (선택된 필터의 공병증을 가진 환자만 표시):
```
┌────────────────────────────────────┐
│ 김철수  45세 남  [초진]             │
│ 현재 HbA1c: 7.8%                  │
│ [비만]                             │
│ "당뇨라고 해서 왔어요."             │
│                         [선택 →]   │
└────────────────────────────────────┘
```

표시 정보: 이름, 나이, 성별, 진료유형 뱃지, 현재 HbA1c, 공병증 태그, 환자 호소 한마디

공병증 태그 색상: 심부전=파랑, CKD=노랑, 비만=초록, 심혈관=빨강, MASH=주황, Stroke=보라

### 3-3. 처방 화면

**화면 구조**: 상단 절반 — 환자/의사 이미지 영역 + 환자 정보 카드 / 하단 절반 — 대화창 + 액션 버튼

**환자 정보 카드** (우상단 고정):
- 이름, 진료유형 뱃지
- BMI
- 현재 HbA1c (빨강, 굵게) / Base HbA1c (회색, 작게)
- 동반 질환 태그 목록

**단계별 UI**:

#### phase: menu (초기)
버튼 2개:
- [처방 조합] → phase: prescribe
- [차트 보기] → phase: chart (환자 상세 정보 텍스트 표시)

차트 표시 내용:
```
[📋 환자 차트 요약]
• 기본정보: 남/45세
• 체중: 82kg / BMI: 26.5
• 진료유형: 초진
• 현재 HbA1c: 7.8% (Base: 7.8%)
• 순응도: 좋음

[🔬 의학적 검사 수치]  ← 수치가 있는 환자에 한함
• eGFR: 45 ml/min/1.73m²
• UACR: 350 mg/g
• LVEF: 35%
• NYHA: 2
• BNP: 50 pg/mL
• NT-proBNP: 150 pg/mL

[🩺 동반 질환]
• 만성신장질환(CKD)

[💊 초기 내원 시 이력]  ← 재진 환자에 한함
• 복용약: 메트포르민 1000mg
```

#### phase: prescribe (약제 선택)
**상병코드 선택** (상단 고정, 필수):
- [E11 당뇨] [I50 심부전] [N18 신장병] 토글 버튼
- 최소 1개 선택 필수 (비약물 처방만인 경우 제외)

**5슬롯 목록**:
```
[급여 1]   [약제명 또는 "처방 약제 비어있음"]  [변경]
[급여 2]   ...                                [변경]
[급여 3]   ...                                [변경]
[본인부담1] ...                               [변경]
[본인부담2] ...                               [변경]
```
- 슬롯 1~3: 파랑 배지 "급여 N"
- 슬롯 4~5: 주황 배지 "본인부담 N"
- [변경] 클릭 → 약제 카테고리 선택 화면

**약제 카테고리 선택**:
- 전체 약제 보기
- 단일제 / 메트포르민 2제 복합제 / 2제 복합제 / 3제 복합제 / 주사제 / 기타/생활습관
- 카테고리 선택 → 해당 약제 목록 (이름 + HbA1c 강하량 표시)
- [슬롯 비우기] 버튼으로 해당 슬롯 초기화

**하단 버튼**: [취소] [처방 확정 →]

#### phase: result (처방 완료)
- 대화창에 처방 결과 요약 텍스트 표시
- [결과 리포트 보기 →] 버튼 → 결과 리포트 화면으로 이동

### 4-1. 환자 데이터 구조

```typescript
interface Patient {
  id: string;              // "p1" ~ "p20"
  name: string;
  age: number;
  gender: 'M' | 'F';
  weight: number;          // kg
  bmi: number;
  initialHba1c: number;   // Base HbA1c (%)
  type: '초진' | '재진' | '리핏';
  desc: string;            // 환자 호소 한마디
  comorbidities: string[]; // 공병증 목록
  adherence: '좋음' | '나쁨';
  order: number;           // 정렬 순서

  // 심부전 관련 (없으면 0)
  lvef: number;            // LVEF (%)
  nyha: number;            // NYHA 등급
  bnp: number;             // BNP (pg/mL)
  ntprobnp: number;        // NT-proBNP (pg/mL)
  hfHospitalization: boolean;  // 12개월 내 입원
  echoAbnormal: boolean;       // 초음파 이상
  hfStandardTx: boolean;       // HF 표준치료 투여 중

  // CKD 관련 (없으면 0)
  egfr: number;            // eGFR (ml/min/1.73m²)
  uacr: number;            // UACR (mg/g)
  dipstick: boolean;       // 요단백 1+ 이상
  ckdStandardTx: boolean;  // CKD 표준치료 투여 중

  // 이전 복용 이력 (재진/리핏용)
  prevDrugs: string[];     // 약제 id 배열 (5슬롯, 없으면 "")
  prevTreatment: string;   // 이전 치료 메모

  imageUrl: string;        // 환자 이미지 URL (없으면 "")
}
```

| 공병증 이름 | 호전 메시지 | 악화 메시지 |
|---|---|---|
| 심부전 | 숨차는 증상도 덜하고 편안해졌어요. | 숨이 차고 가슴이 답답해요. |
| 만성신장질환(CKD) | 소변 거품도 줄고 붓기도 덜하네요! | 소변 보기가 불편하고 더 붓는 것 같아요. |
| 비만 | 체중이 줄어서 몸이 가볍습니다! | 살이 더 찌는 것 같아 걱정이에요. |
| 심혈관 | 심장 건강이 좋아진 것 같아 안심입니다. | 심장 쪽이 왠지 모르게 불편합니다. |
| MASH | 간 수치도 좋아지고 피로감도 덜합니다. | 간 쪽에 무리가 가는 건 아닐지 걱정돼요. |
| Stroke | 뇌졸중 예방도 된다니 마음이 놓입니다. | 머리 쪽이 가끔 어지러운 것 같아요. |
| 위장장애 | 속이 편안해서 다행입니다. | 약 먹고 속이 더부룩하고 소화가 안 돼요. |
| 생식기감염 | 특별한 감염 증상 없이 깨끗합니다. | 소변 볼 때 불편하고 가렵습니다. |
| 저혈당 | 저혈당 없이 안정적으로 유지되고 있어요. | 가끔 식은땀이 나고 손이 떨려요. |
| 전반적 개선 | 전체적인 컨디션이 몰라보게 좋아졌습니다! | 아직 전체적인 컨디션은 잘 모르겠습니다. |

### 4-6. 약제 데이터 구조

```typescript
interface Medication {
  id: string;              // "m_1" ~ "m_60"
  name: string;
```

## 5. 처방 계산 로직

처방 확정 버튼 클릭 시 아래 순서로 계산한다.

### 5-1. 현재 환자 상태 계산 (getPatientCurrentState)

이전 처방 이력이 없는 환자(초진):
```
currentHba1c = initialHba1c - (prevDrugs에 해당하는 약제 effect 합산)
currentWeight = weight
currentLvef = lvef
currentNyha = nyha
currentBnp = bnp
currentNtprobnp = ntprobnp
currentEgfr = egfr
currentUacr = uacr
```

이전 처방 이력이 있는 환자(재진/리핏, 같은 앱 내에서 이전에 처방한 경우):
```
currentHba1c = 가장 최근 처방 로그의 newHba1c
currentWeight = 가장 최근 처방 로그의 newWeight
(나머지 지표도 동일)
```

### 5-2. 처방 효과 계산

```
선택된 약제 배열: selectedDrugs (null 제외)
활성 계열 목록: 모든 선택 약제의 classes 합집합

총효과 초기값: { h:0, w:0, l:0, n:0, b:0, nt:0, eg:0, ua:0 }

각 약제에 대해:
  1. eH = d.effect (비약물이면 0)
  2. 부작용 계산:
     - 위장장애 계열 약제이고, 환자의 과거 처방에서 위장장애 2회 이상 발생 → 부작용 스킵
     - sideEffectExemptions 설정에 해당하는 조합이면 → 부작용 스킵
     - 위 조건 미해당 시: Math.random() * 100 < sideEffectProb 이면 부작용 발생
       → sideEffects 배열에 "[약제명] {sideEffectMsg}" 추가
       → eH = max(0, eH - sideEffectPenalty)
  3. 총효과에 각 수치 누적
  4. beneficialComorb 중 환자 공병증에 포함된 것 → 호전 메시지 수집
  5. worseningComorb 중 환자 공병증에 포함된 것 → 악화 메시지 수집

병포장 보너스:
  - 선택된 모든 약제(비약물 제외)의 pkg가 'bottle'이면 iAP = true
  - iAP = true이면: 총효과.h += packagingBonusEffect (기본값 0.3)

순응도 나쁨 처리:
  - 환자 adherence === '나쁨' AND iAP = false AND 비약물 아님
  → 총효과.h = -0.4 (HbA1c 오히려 상승)
```

### 5-3. 최종 수치 계산

```
newHba1c = max(4.5, currentHba1c - 총효과.h, 소수점 1자리)
newWeight = currentWeight + 총효과.w
newLvef = currentLvef ? currentLvef + 총효과.l : ""
newNyha = currentNyha ? currentNyha + 총효과.n : ""
newBnp = currentBnp ? currentBnp + 총효과.b : ""
newNtprobnp = currentNtprobnp ? currentNtprobnp + 총효과.nt : ""
newEgfr = currentEgfr ? currentEgfr + 총효과.eg : ""
newUacr = currentUacr ? currentUacr + 총효과.ua : ""
```

### 5-4. 처방 결과 메시지 조합

**비약물 처방만인 경우**:
```
환자 피드백: msgLifestyle ("추천해주신 대로 생활습관을 바꾸니 몸이 가볍습니다.")
```

**부작용 발생한 경우**:
```
환자 피드백: msgSideEffect ("선생님, 약을 먹고 나서 속이 좀 불편해요.")
공병증 메시지 미표시
```

**정상 처방인 경우**:
```
환자 피드백: 병포장이면 msgPackaging, 아니면 msgSuccess
공병증 메시지: 호전 메시지 + 악화 메시지 (중복 제거)
```

**순응도 나쁨인 경우**:
```
"환자가 약을 제대로 먹지 않았습니다... 혈당 상승!" 표시
```

**규칙 1. DPP-4i + GLP-1 RA 병용 금지**
- 보험 처방(슬롯 1~3) 중 isInsuranceException=false, isNotDrug=false인 약제
- 해당 약제들의 classes에 dc_dpp4와 dc_glp1가 동시에 존재하면 삭감

**규칙 2. 당뇨(E11) 상병 시 처방 기준**
- 상병코드에 E11 포함 + isInsuranceException=false + isNotDrug=false 약제가 1개 이상
- HbA1c < 6.5% → "당뇨(E11) 초기 HbA1c 6.5% 미만 처방 삭감!"
- HbA1c >= 6.5%이고 단일 계열이며 메트포르민(dc_met) 미포함 → "1차 메트포르민 미사용 삭감!"
- HbA1c >= 6.5%이고 2제 이상이며 HbA1c < 7.5% → "초기 급여 2제 병용 기준 미달 삭감!"
- HbA1c >= 7.5%이고 2제 이상이며 메트포르민(dc_met) 미포함 → "병용 요법 1차약제 미포함 삭감!"

**eGFR 하한선 자동 체크** (선택 화면에서 경고 표시 또는 선택 불가 처리):
- 약제의 egfrLimit > 0이고 환자의 currentEgfr < egfrLimit이면 처방 불가 또는 경고

## 7. 비당뇨 환자 보험 적용 조건

환자의 initialHba1c < 6.5인 경우 비당뇨 환자로 간주.
비당뇨 환자에게 SGLT-2i 처방 시 아래 조건을 충족해야 보험 적용:

### HFrEF (LVEF < 40) 특례: allowHFrEFCoverage = true인 약제
- LVEF ≤ Admin설정 hfLvefMax (기본 40) AND
- NYHA ≥ Admin설정 hfNyhaMin (기본 2) AND
- (BNP ≥ 35 OR NT-proBNP ≥ 125) AND
- hfStandardTx = true AND
- (hfHospitalization = true OR echoAbnormal = true)

### HFpEF (LVEF ≥ 40) 특례: allowHFpEFCoverage = true인 약제
- 위와 동일 조건 (LVEF ≥ 40인 환자)

### CKD 특례: allowCkdCoverage = true인 약제
- eGFR ≥ Admin설정 ckdEgfrMin (기본 20) AND eGFR ≤ Admin설정 ckdEgfrMax (기본 75) AND
- UACR ≥ Admin설정 ckdUacrMin (기본 200) AND
- dipstick = true AND
- ckdStandardTx = true

## 9. Admin 콘솔

비밀번호(기본 1024) 입력으로 접근. 탭 구조:

| 탭 | 기능 |
|---|---|
| 설정 | 로그인 화면 색상, 타이틀, 로고 이미지, 처방 기준값, 메시지 텍스트 수정 |

| 삭감 규칙 | 병용 금지 계열 조합 추가/삭제 |
| 허용 조합 | 삭감 예외 조합 추가/삭제 |
| 부작용 면제 | 특정 계열 조합 처방 시 부작용 면제 설정 |
| 세션 이력 | 병원/의사별 처방 이력 조회, CSV 내보내기, 전체 초기화 |

## 12. 미결 사항 (의사결정 필요)

1. **세션 이력 누적**: 같은 병원+의사 재방문 시 이전 처방 이력 이어받기 vs 매번 새로 시작
2. **약제 eGFR 하한선 초과 시 처리**: 선택 자체를 막을지 vs 경고만 표시할지
3. **결과 리포트 공유**: PDF 저장 또는 링크 공유 기능 필요 여부
4. **브랜딩**: 제약사 로고·컬러 고정 vs Admin에서 커스터마이즈
5. **환자 직접 추가**: 담당자가 실제 케이스 기반으로 환자를 현장에서 생성하는 기능 필요 여부
